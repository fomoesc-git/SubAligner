# SubAligner — 配音文案字幕生成器

## 项目信息

| 项目 | 内容 |
|------|------|
| **名称** | SubAligner |
| **类型** | 本地桌面应用（Tauri 2.x） |
| **版本** | v1.0.2 |
| **核心功能** | 导入配音音频 + 粘贴配音文案 → AI 自动生成精确的 SRT 字幕文件，全程无需联网 |
| **GitHub** | github.com/fomoesc-git/SubAligner |
| **设计署名** | 本工具由@常宁千影设计 |
| **开发环境** | macOS (M系列芯片) |

## 技术架构

### 技术栈

| 层级 | 技术 |
|------|------|
| 桌面框架 | Tauri 2 |
| 前端 | Vue 3 + TypeScript + Naive UI |
| AI 引擎 | Python FastAPI (Sidecar 进程) |
| 强制对齐 | Silero VAD + wav2vec2 CTC |
| 音频处理 | ffmpeg |

### 通信架构

```
用户操作 (Vue UI)
    │
    ├─ Tauri invoke ──→ Rust (start_engine) ──→ 启动 Python Sidecar 进程
    │                                                    │
    ├─ HTTP 127.0.0.1:{port}/api/* ◄────────────────────┘
    │
    ├─ 引擎崩溃 → Rust emit("engine-exit") → 前端监听 → 显示重试按钮
    └─ 引擎错误 → Rust emit("engine-error") → 前端监听
```

- Sidecar 通过 `portpicker` 动态分配端口，前端通过 Tauri invoke 获取端口号后发 HTTP 请求
- 单 worker（torch 线程安全限制）

### AI 对齐流程

1. `TextProcessor.split_into_sentences()` — 按换行+标点分句（换行=硬边界）
2. `AudioProcessor.preprocess()` — ffmpeg 转 16kHz mono WAV
3. `SileroVAD.segment()` — VAD 语音分段
4. `AlignmentEngine._assign_sentence_timings()` — 按字数比例分配时间
5. `AlignmentEngine._energy_distribute()` — 基于音频能量分布字符时间戳
6. `_group_subtitles_by_sentences()` — 每句一条字幕

**关键设计**：短句子永不在中间被切断；句子边界始终被尊重。

## 当前已知问题与待解决事项

### 🔴 关键问题：新设备引擎启动失败（v1.0.2 已修复，未验证）

**现象**：在开发机正常，但在新安装的 Mac/Windows 设备上拖入音频报错：`无法读取音频：网络请求失败(/api/audio/info):load failed`

**v1.0.2 修复内容**：
1. macOS Gatekeeper 隔离属性移除（`xattr -cr`）
2. 引擎健康检查通过后才设置端口和 ready 状态
3. 启动超时延长至 60 秒
4. ffmpeg 执行权限自动修复（`_ensure_executable()`）
5. 引擎崩溃后重试机制

**验证状态**：已推送到 GitHub，尚未在新设备上验证。

### 🟡 待关注问题

1. **模型首次下载**：wav2vec2 模型 ~350MB 首次使用时需下载。中国网络自动切换 ModelScope 镜像。UI 中没有明显的模型下载进度提示。
2. **PyInstaller onefile 启动慢**：每次启动都要解压 ~500MB 临时文件，可能需要 30-60 秒。可考虑改为 `--onedir` 模式。
3. **Sidecar 进程未在退出时清理**：Rust 侧的 `_child` 变量被丢弃，没有在应用退出时主动 kill sidecar 进程。

## 开发与构建

### 本地启动

```bash
cd SubAligner

# 安装前端依赖
npm install

# 首次需要构建 Python Sidecar
cd engine
pip install -r requirements.txt
pip install pyinstaller
python build.py
cd ..

# 启动 Tauri 开发模式
npm run tauri dev
```

### 单独启动 Python 引擎（调试用）

```bash
cd engine
pip install -r requirements.txt
python main.py --port 9580
```

### 构建 Release

```bash
npm run tauri build
```

### 发版流程

1. 修改三处版本号：`package.json` + `src-tauri/tauri.conf.json` + `src-tauri/Cargo.toml`
2. 同步 `src/components/AppHeader.vue` 中的硬编码版本号
3. `git add && git commit && git push origin main`
4. 等 CI 构建完成 → 去 GitHub Releases 页面 → Publish release

## 关键设计决策

1. **不用 Whisper ASR**：因为已有用户提供的文案，只需做强制对齐（wav2vec2 CTC），不需要语音识别
2. **不用 ASS/VTT 导出**：只导出 SRT 字幕
3. **不做批量处理**：单文件处理模式
4. **VAD 不对称边距**：`margin`(0.12s) 保护上句尾部，`margin_head`(0.22s) 保护下句起音
5. **暗色模式**：根据时间自动切换（20:00-6:00），用户可手动覆盖
6. **橙色主题**：`#e8792b` / `#f5a623`
7. **生成按钮无描边**：用 `:deep()` 覆盖 Naive UI 的 `.n-button__border` 和 `.n-button__state-border`
8. **SRT 无序号**：导出的 SRT 和复制的内容都不含数字编号
9. **编辑功能**：字幕时间和文本均可点击编辑，修改会反映到导出
10. **跨平台路径处理**：Python 用 `pathlib.Path`，TS 用 `path.join()`
11. **换行符**：源码 LF，SRT 输出 CRLF
12. **Sidecar 通信**：HTTP localhost（非 Unix Socket）
13. **临时文件**：只用 ASCII 文件名
